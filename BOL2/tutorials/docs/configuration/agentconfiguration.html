The Agent configuration file contains configuration parameters identifying the name of the agent, the agent implmentation class to use, and the components to be loaded into the Agent. An Agent Configuration file needs to be named [AGENT_NAME].ini and match the agent name with the one present in the Node configuration file. Here is an example Agent Configuration file for an Agent called Foo: 
			[ Cluster ]
			class = org.cougaar.core.agent.SimpleAgent
			MessageAddress = Foo
			

			[ PlugIns ]
			plugin = org.cougaar.core.servlet.SimpleServletComponent(org.cougaar.planning.servlet.PlanViewServlet, /tasks)
			plugin = org.cougaar.core.servlet.SimpleServletComponent(org.cougaar.planning.servlet.CompletionServlet, /completion)

Lets go over the example line by line:
1. [Cluster] This line identifies that this portion of the configuration file contains properties of the agent.
2. class = org.cougaar.core.agent.SimpleAgent This line identifies the implementation class to use for the Agent. Most of the time, unless you would like to change an Agent's behaviour, you should use SimpleAgent.
3. MessageAddress = Foo This line identifies the name of the Agent
4. [ Plugins ] This line indentifies that this portion of the configuration file contains components to be loaded into the agent.
5. plugin = org.cougaar.core.servlet.SimpleServletComponent(org.cougaar.planning.servlet.PlanViewServlet, /tasks) This line identifies a component to be loaded into the Foo Agent. The plugin's component is org.cougaar.core.servlet.SimpleServletComponent. The values inside the () are used to pass argument(s) to the component. In this case two arguments are passed to SimpleServletComponent.
This is the basics of the Agent Configuration File, the next step is to configure Agent Relationships.
