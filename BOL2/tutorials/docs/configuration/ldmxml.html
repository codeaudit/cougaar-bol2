The LDM XML File is used to create prototypes and to also create instances of Assets on an Agent's Blackboard during the starup of a Cougaar Agent. The org.cougaar.lib.plugin.UTILLdmXMLPlugin plugin is a utility plugin that, when placed in an agent, will try to parse a ldm.xml file named with the following naming convention: [AGENT_NAME].ldm.xml.
Below is a sample LDM XML file for the Foo agent called Foo.ldm.xml: 
			<?xml version="1.0"  encoding="US-ASCII"?>
			<!DOCTYPE ClusterAssets SYSTEM "AssetList.ldm.dtd" []>

			<AssetList>
			  <prototype name="FooAsset">
			  	<object class="com.foo.asset.FooAsset" id = "foo-proto">
			  		<field name="TypeIdentificationPG" type="object">
				        <value>
				          	<object class="org.cougaar.planning.ldm.asset.NewTypeIdentificationPG">
					          <field name="TypeIdentification" type="String">
					            <value>ObjectAsset</value>
					            </field>
					          <field name="Nomenclature" type="String">
            					<value>ObjectAsset</value>
          					  </field>
					          <field name="AlternateTypeIdentification" type="String">
					            <value>ObjectAsset</value>
					          </field>
			          		</object>
			        	</value>
			      	</field>
			  	</object>
			  </prototype>
		 
			 <instance prototype="FooAsset" id="Foo Asset 1">
  				 <schedule end="01/05/11" start="06/11/03"/>
			 </instance>
		   	<instance prototype="PackerAsset" id="Foo Asset 2">
			  	<schedule end="01/05/11" start="06/11/03"/>
			  </instance>
   			</AssetList>
This example will cache a prototype of the FooAsset in the PrototypeRegistry Service under the name FooAsset (The name attribute of the prototype element). The LDM Plugin will also create two instances of the FooAsset prototype and set their ItemIdentifications to "FooAsset1" and "FooAsset2".
